# Rencana Fitur

> Dokumentasikan minimal **5 fitur utama** proyek Anda.
> Salin dan ulangi blok di bawah untuk setiap fitur tambahan.

---

## Fitur 1 — [Login]

**Role Penanggung Jawab:** `[Frontend | Backend | DevOps]`

**Sumber Data:** `[Internal System + Third-Party API — Google OAuth / Facebook Login]`

**Deskripsi & Ekspektasi:**
`[Fitur login memungkinkan pengguna membuat akun dan masuk ke platform Rencana Liburan menggunakan email & password, atau melalui akun Google/Facebook (Single Sign-On). Alur kerja: pengguna memasukkan kredensial → sistem memverifikasi ke database → token autentikasi JWT dibuat → pengguna diarahkan ke halaman dashboard.]`

## Fitur 2 — [Pencarian]

**Role Penanggung Jawab:** `[Frontend | Backend]`

**Sumber Data:** `[Internal System + Third-Party API — Google Places API / Amadeus Travel API]`

**Deskripsi & Ekspektasi:**
`[Fitur pencarian memungkinkan pengguna mencari destinasi wisata, paket liburan, hotel, dan aktivitas berdasarkan kata kunci, lokasi, rentang tanggal, jumlah peserta, dan anggaran. Sistem dilengkapi dengan autocomplete saat mengetik dan filter lanjutan (kategori wisata, rating, harga, fasilitas).]`

---

## Fitur 3 — [fitur like dan dislike]

**Role Penanggung Jawab:** `[Frontend | Backend]`

**Sumber Data:** `[Internal System]`

**Deskripsi & Ekspektasi:**
`[Pengguna yang sudah login dapat memberikan reaksi like (👍) atau dislike (👎) pada konten seperti destinasi wisata, paket liburan, atau ulasan perjalanan. Data reaksi ini digunakan oleh sistem untuk mempersonalisasi rekomendasi liburan sesuai preferensi pengguna.]`

---

## Fitur 4 — [Komen]

**Role Penanggung Jawab:** `[Frontend | Backend]`

**Sumber Data:** `[Internal System]`

**Deskripsi & Ekspektasi:**
`[Fitur komentar memungkinkan pengguna berbagi pengalaman, tips perjalanan, dan ulasan pada halaman destinasi atau paket liburan. Mendukung balasan komentar bertingkat (threaded comments), laporan komentar spam/tidak pantas, dan moderasi konten oleh admin.]`

---

## Fitur 5 — [chat dengan agent dari website]

**Role Penanggung Jawab:** `[Frontend | Backend]`

**Sumber Data:** `[Third-Party API — Anthropic Claude API]`

**Deskripsi & Ekspektasi:**
`[Pengguna dapat berinteraksi langsung dengan AI Agent melalui widget chat yang tersedia di setiap halaman website. Agent bertugas membantu pengguna mendapatkan rekomendasi destinasi liburan, menjawab pertanyaan seputar tempat wisata, menyusun itinerary perjalanan, serta memberikan estimasi anggaran liburan.]`
